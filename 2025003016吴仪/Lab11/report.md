## 一、应用功能与展示信息

## 二、布局结构说明
本次界面采用**模块化、分层嵌套**的结构设计，各组件职责清晰，布局逻辑明确：

1.  **根布局**：使用 `Scaffold` 作为页面根容器，填充整个屏幕空间，适配系统状态栏与导航栏，提供稳定的页面框架。
    - 顶部集成 `TopAppBar` 导航栏，根据页面状态动态显示标题与返回按钮
    - 底部为主要内容区域，根据设备尺寸加载不同布局内容

2.  **主内容布局**：根据设备屏幕宽度自动切换两种布局模式：
    - **小屏模式（手机）**：采用 `LazyColumn` 实现可滚动的单列表布局，点击项目跳转至详情页，支持返回导航
    - **大屏模式（平板/宽屏）**：采用 `Row` 水平布局，实现**左侧列表+右侧详情**双栏并排展示，点击列表项同步更新右侧内容

3.  **列表卡片容器**：使用 `Card` 组件实现列表项样式，具备默认圆角与阴影效果，提升界面视觉层次。
    - 卡片内边距统一设置，保证点击区域充足
    - 卡片之间设置固定间距，避免布局拥挤

4.  **卡片内部结构**：采用 `Row` 水平布局，将图标区域与文字区域左右排列：
    - 左侧图标区域：固定尺寸图标，直观展示运动项目标识，统一尺寸与边距
    - 右侧文字区域：使用 `Column` 垂直布局，依次展示运动名称、项目描述信息，文本样式区分标题与说明

5.  **详情页布局**：采用 `Column` 垂直布局，依次展示运动名称、项目描述、详细介绍文本，排版清晰易读。
    - 标题文本使用大字号突出显示
    - 描述与详情文本使用不同字号区分层级
    - 内容区域设置统一内边距，保证阅读体验

6.  **导航栏布局**：使用 `TopAppBar` 实现顶部导航栏，根据当前页面状态动态显示/隐藏返回按钮，适配不同布局模式。
    - 小屏详情页：显示返回按钮，支持返回列表页
    - 大屏模式：不显示返回按钮，界面保持简洁

## 三、核心实现原理
1.  **窗口尺寸适配**：通过 `WindowWidthSizeClass` 识别设备屏幕宽度，自动判断当前设备应采用的布局模式，实现一次开发多设备适配。
    - `Compact` 宽度（手机）：采用单列表+详情页模式
    - `Expanded` 宽度（平板/宽屏）：采用双栏并排模式

2.  **状态管理**：使用 `remember` + `mutableStateOf` 管理当前选中的运动项目，监听状态变化自动刷新界面，实现页面切换与详情同步更新。
    - 小屏模式：状态变化触发页面跳转
    - 大屏模式：状态变化触发右侧详情内容更新

3.  **懒加载列表**：使用 `LazyColumn` 实现列表懒加载，仅渲染屏幕可见项，提升列表滑动性能，适配大量数据展示场景。
    - 支持无限滚动查看所有运动项目
    - 列表项复用机制保证流畅度

4.  **安全调用与非空处理**：通过 `?.let` 安全调用处理可空类型，避免空指针异常，保证界面稳定运行。
    - 详情页加载时的非空判断
    - 状态更新时的安全访问

5.  **返回键处理**：通过 `BackHandler` 组件处理系统返回键事件，根据当前布局模式执行不同逻辑。
    - 小屏详情页：返回列表页
    - 大屏模式：无页面跳转，保持当前布局

## 六、实验中遇到的问题与解决过程
1.  **问题**：`WindowWidthSizeClass` 无法识别 `Expanded` 常量，编译报错。  
    **解决**：检查依赖是否添加 `material3-window-size-class`，并确保在代码中使用完整类名 `WindowWidthSizeClass.Expanded`。

2.  **问题**：在详情页访问 `currentSport` 属性时，报 `Smart cast to 'Sport' is impossible` 错误。  
    **解决**：使用 `?.let` 安全调用，在 `let` 块内访问属性，或在访问前进行非空判断。

3.  **问题**：大屏模式下，列表项点击后右侧详情不更新。  
    **解决**：确保 `currentSport` 使用 `mutableStateOf` 声明，并在点击事件中正确更新 `currentSport` 的值，触发重组。

4.  **问题**：应用在不同设备上的字体和图标大小显示不一致。  
    **解决**：统一使用 `MaterialTheme.typography` 提供的文本样式，并为图标设置固定的 `dp` 尺寸，确保在不同分辨率下显示一致。
