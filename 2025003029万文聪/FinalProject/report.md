## 2. App 名称、选题背景和目标用户

App 名称：Study Planner 学习计划助手。

我选择做学习任务与课程作业管理 App，主要是因为平时课程、作业、实验和复习任务比较分散，如果只靠临时记事，很容易忘记截止时间。本应用面向大学生或需要管理多门课程任务的用户，用来集中记录课程、作业、复习任务和完成情况。

## 3. 核心功能说明

本项目主要实现了以下功能：

- 查看所有学习任务。
- 新增、编辑、删除任务。
- 查看任务详情。
- 切换任务完成状态。
- 按关键词搜索任务或课程。
- 按截止时间或优先级排序。
- 显示或隐藏已完成任务。
- 在首页显示学习进度统计，包括全部任务、待完成、已完成和高优先级任务。
- 新增课程，并查看课程对应的任务。
- 从在线接口加载推荐学习任务，并添加到本地任务列表。
- 在设置页保存显示偏好、排序偏好和主题偏好。

## 4. 页面结构说明

应用主要包含以下页面：

- 首页：显示任务列表，支持搜索、排序、筛选、完成状态切换和进度统计。
- 课程页：显示课程列表，可以新增课程，并查看某门课程下的任务。
- 任务详情页：显示任务标题、课程、说明、截止时间、优先级和完成状态。
- 新增 / 编辑任务页：填写或修改任务内容。
- 在线推荐页：从网络接口加载任务模板，并加入本地任务。
- 设置页：保存是否显示已完成任务、排序方式和主题偏好。

## 6. Room 数据库设计

数据库包含两张主要表：课程表和任务表。

### courses 表

字段说明：

- id：课程主键，自增。
- name：课程名称。
- teacher：教师名称。
- classroom：上课地点。
- color_hex：课程颜色。
- created_at：创建时间。

### study_tasks 表

字段说明：

- id：任务主键，自增。
- course_id：所属课程 ID，对应 courses 表中的 id。
- title：任务标题。
- description：任务说明。
- deadline：截止日期。
- priority：优先级。
- is_done：是否完成。
- created_at：创建时间。

两张表之间通过 course_id 建立关联，一个课程可以对应多个学习任务。

主要 DAO 功能包括：

- 查询全部课程。
- 查询全部任务。
- 按课程查询任务。
- 根据关键词搜索任务标题或课程名称。
- 按截止时间排序。
- 按优先级排序。
- 插入、更新、删除任务。

## 7. DataStore 设计

DataStore 用来保存用户偏好，主要包括：

- 是否显示已完成任务。
- 默认排序方式。
- 是否启用深色模式。
- 最近一次搜索内容。

这些数据会在 App 启动后读取，并影响首页任务列表和主题显示。设置页修改后，重新打开 App 仍然可以保留上次的选择。

## 8. 网络功能设计

本项目的网络功能使用 Retrofit 实现，接口地址为：

```text
https://jsonplaceholder.typicode.com/todos?_limit=12
```

接口返回的数据会转换成适合学习场景的推荐任务模板，并显示在“在线推荐任务”页面。用户可以选择课程后，把推荐任务添加到本地任务列表中。这样网络数据不是单独展示，而是参与了“推荐任务 → 保存到本地 Room → 首页显示”的完整流程。

该接口是公开测试接口，不需要 API key，项目中也没有写入真实密钥或 token。

## 9. ViewModel 和 UiState 设计

项目使用 ViewModel 管理界面状态和业务逻辑。Composable 只负责显示界面和触发事件，不直接访问数据库或网络。

主要 UiState 包括：

- HomeUiState：表示首页加载、成功和错误状态。
- CoursesUiState：表示课程列表状态。
- RecommendedUiState：表示在线推荐任务的加载、成功和错误状态。
- EditTaskUiState：表示新增或编辑任务页面的表单状态。

ViewModel 会组合 Room Flow、DataStore Flow 和搜索关键词，并把最终状态暴露给 UI。

## 10. Repository 设计

项目使用 StudyPlannerRepository 隔离数据来源。ViewModel 不直接调用 DAO 或 Retrofit，而是通过 Repository 访问数据。

Repository 主要负责：

- 调用 CourseDao 和 TaskDao 操作本地 Room 数据库。
- 调用 Retrofit 服务加载在线推荐任务。
- 首次启动时初始化部分课程和任务数据。
- 把在线推荐任务保存到本地任务表。

这样的结构可以让本地数据、网络数据和界面状态分开管理，代码更容易维护。

## 12. 遇到的问题与解决过程

问题一：一开始我只考虑用一张任务表保存所有内容，但这样每个任务都要重复保存课程名称、教师和地点，不方便管理。后面改成了课程表和任务表两张表，通过 course_id 关联，这样课程信息可以单独维护，任务也能按课程查询。

问题二：网络功能开始只是准备单独展示在线数据，但这样和本地任务管理关系不大。后来改成“在线推荐任务可以加入本地任务列表”，这样网络请求、Repository、Room 数据库存储和首页列表就能连成一个完整流程。

问题三：设置页如果只存在内存里，重新打开 App 后会丢失。解决方式是使用 DataStore 保存显示已完成任务、排序方式和主题偏好，让用户设置能够持久保存。

问题四：调试时模拟器有时会停留在之前实验运行过的旧应用界面，所以需要确认当前安装和启动的是 `com.example.studyplanner`，避免截图截成旧项目页面。

## 15. 运行说明

运行步骤：

1. 使用 Android Studio 打开项目根目录 `StudyPlanner_FinalProject`。
2. 等待 Gradle Sync 完成。
3. 选择模拟器或真机运行。
4. 首页可以查看和管理学习任务。
5. 在线推荐页需要网络权限，用于加载推荐任务。

最低 Android 版本：minSdk 24。

特殊权限：

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

## 16. 项目亮点与未来改进方向

本项目把课程和任务拆成两张表，首页可以直接看到每个任务对应的课程。在线推荐任务不是单独展示，而是可以添加到本地任务列表里，所以网络功能和本地数据库是连接在一起的。设置页保存了筛选和主题偏好，重新打开 App 后还能保留上次的选择。首页还加入了学习进度统计，方便快速了解当前任务完成情况。

后续可以继续改进的地方：

- 增加本地通知提醒。
- 增加任务完成统计图。
- 增加日历视图。
- 支持导出任务数据。
